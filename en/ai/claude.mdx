Claude Desktop supports MCP servers natively. With the Vendaze MCP Server configured, you can ask Claude questions about your CRM data and request it to perform actions directly, without opening Vendaze.

## Prerequisites

- Claude Desktop installed (version with MCP support)
- Valid Vendaze access token (obtained via OAuth 2.1 flow)

## Configure the MCP Server

Edit the Claude Desktop configuration file:

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`  
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Add the Vendaze server:

```json
{
  "mcpServers": {
    "vendaze": {
      "url": "https://api.vendaze.com/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_ACCESS_TOKEN"
      }
    }
  }
}
```

Restart Claude Desktop. The tools icon in the chat will indicate the MCP server is connected.

## Getting the access token

If you do not have an access token yet, follow the [first request guide](/en/guides/first-request) to register an app and complete the OAuth flow.

For personal use with Claude Desktop, we recommend registering an app with broad scopes and generating a long-lived token via refresh token.

## What you can do

With MCP configured, you can talk to Claude in natural language:

**Queries:**

- "What are the 10 highest-value deals in the Commercial pipeline?"
- "Show me the contacts at Acme Corp with all their information."
- "Which tasks are open with an overdue date?"
- "List the activities on deal ID abc-123 from the last two weeks."

**Actions:**

- "Create a contact named John Smith, at company XYZ, with email john@xyz.com."
- "Move the deal 'Quarterly Contract' to the 'Proposal Sent' stage."
- "Add a note on contact Ana Costa saying she returns on the 15th."
- "Create a task to call the customer tomorrow at 2pm, linked to deal ID xyz-456."

## Token renewal

Access tokens expire in 1 hour. To avoid interruptions when using Claude, create a script that renews the token automatically and updates the configuration file:

```javascript
// renew-token.js — run before using Claude Desktop
const fs = require('fs');
const path = require('path');

async function renewToken() {
  const res = await fetch('https://api.vendaze.com/oauth/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      grant_type: 'refresh_token',
      refresh_token: process.env.VENDAZE_REFRESH_TOKEN,
      client_id: process.env.VENDAZE_CLIENT_ID,
      client_secret: process.env.VENDAZE_CLIENT_SECRET,
    }),
  });

  const { access_token } = await res.json();

  const configPath = path.join(
    process.env.HOME,
    'Library/Application Support/Claude/claude_desktop_config.json',
  );

  const config = JSON.parse(fs.readFileSync(configPath, 'utf-8'));
  config.mcpServers.vendaze.headers.Authorization = `Bearer ${access_token}`;
  fs.writeFileSync(configPath, JSON.stringify(config, null, 2));

  console.log('Token renewed successfully.');
}

renewToken();
```

<Note>
  Store the `refresh_token` in a secure location, such as system environment variables or a password
  manager. Never in plain text in git-tracked files.
</Note>
