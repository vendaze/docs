O GitHub Copilot no VS Code suporta servidores MCP a partir da versão 1.99 do editor, com o modo de agente habilitado. Com o MCP Server da Vendaze configurado, o Copilot pode consultar e manipular dados do CRM diretamente durante uma sessão de desenvolvimento.

## Pré-requisitos

- VS Code versão 1.99 ou superior
- GitHub Copilot ativo na conta
- GitHub Copilot Chat habilitado
- Access token válido da Vendaze

## Configurar o MCP Server

Adicione o servidor Vendaze às configurações do VS Code. Você pode fazer isso via interface ou editando o `settings.json` diretamente:

```json
{
  "github.copilot.chat.mcp.servers": {
    "vendaze": {
      "url": "https://api.vendaze.com/mcp",
      "headers": {
        "Authorization": "Bearer SEU_ACCESS_TOKEN"
      }
    }
  }
}
```

Para abrir o `settings.json`: `Cmd/Ctrl + Shift + P` > **Preferences: Open User Settings (JSON)**.

Após salvar, reinicie o VS Code. No painel do Copilot Chat, o ícone de ferramentas mostrará os servidores MCP disponíveis.

## Usar no Copilot Chat

Abra o Copilot Chat com `Cmd/Ctrl + Shift + I` e use o modo **Agent** (selecione o ícone de agente). Com o agente ativo, o Copilot pode chamar ferramentas do MCP Server automaticamente quando relevante para a conversa:

**Exemplos de perguntas:**

- "Quais negócios estão no stage 'Proposta Enviada' com valor acima de R$ 50.000?"
- "Busca o contato com email joao@acme.com e me mostra todos os campos."
- "Cria uma atividade do tipo 'call' no negócio ID xyz-456 com a nota 'Ligação realizada, aguarda retorno'."

**Durante o desenvolvimento:**

- "Estou implementando o endpoint de criação de negócios. Checa os campos disponíveis no workspace para sugerir o body de exemplo."
- "O meu teste está falhando ao criar um contato. Busca um contato real e compara com o payload que estou enviando."

## Usar com comentários de código

Você pode referenciar ferramentas MCP diretamente em comentários enquanto escreve código:

```typescript
// Copilot: usa o MCP da Vendaze para listar os campos adicionais disponíveis
// e gerar os tipos TypeScript correspondentes para custom_fields
```

O Copilot buscará os dados reais via MCP e gerará os tipos baseados na resposta da API.

## Configuração por workspace

Para projetos de integração com a Vendaze, adicione um arquivo `.vscode/settings.json` ao repositório com a configuração do MCP (sem o token):

```json
{
  "github.copilot.chat.mcp.servers": {
    "vendaze": {
      "url": "https://api.vendaze.com/mcp"
    }
  }
}
```

Cada desenvolvedor do time adiciona o header de autorização nas suas configurações pessoais de usuário do VS Code, mantendo os tokens fora do repositório.

## Renovação de token

Tokens de acesso expiram em 1 hora. Crie uma tarefa no `package.json` para renovar automaticamente:

```json
{
  "scripts": {
    "renew-mcp": "node scripts/renew-vendaze-token.js --target vscode"
  }
}
```

O script de renovação atualiza o `settings.json` do VS Code com o novo token. Veja o padrão no [guia do Claude](/pt/ai/claude#renovação-de-token).

<Note>
  O suporte a MCP no Copilot requer que o modo de agente esteja habilitado no Copilot Chat.
  Verifique nas configurações do GitHub Copilot se o modo de agente está ativo.
</Note>
