O Cursor suporta servidores MCP, permitindo que o assistente de IA do editor acesse dados reais do seu CRM enquanto você desenvolve. Isso é especialmente útil para inspecionar dados de produção, testar queries e depurar a sua integração sem sair do editor.

## Configurar o MCP Server no Cursor

Abra as configurações do Cursor e navegue até **Features > MCP Servers** ou edite diretamente o arquivo `~/.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "vendaze": {
      "url": "https://api.vendaze.com/mcp",
      "headers": {
        "Authorization": "Bearer SEU_ACCESS_TOKEN"
      }
    }
  }
}
```

Após salvar, o Cursor recarregará automaticamente os servidores MCP disponíveis.

## Como usar no editor

Com o MCP configurado, o assistente do Cursor (Cmd/Ctrl + L) pode chamar ferramentas do Vendaze durante uma conversa:

**Inspecionar dados reais:**

- "Busca os 5 negócios mais recentes do workspace e me mostra a estrutura de resposta."
- "Qual é o UUID do pipeline principal? Preciso hardcodar para o teste."
- "Listar os campos adicionais disponíveis para pessoas."

**Depurar sua integração:**

- "O código está criando um negócio sem `currency_id`. Checa se esse campo é obrigatório consultando a API."
- "Busca a pessoa com ID abc-123 e compara com o tipo que definimos em `src/types/api.ts`."

**Gerar dados de teste:**

- "Cria 3 contatos de teste com nomes e emails fictícios para eu usar nos testes de integração."

## Usar com `.cursor/rules`

Adicione um arquivo `.cursor/rules` no seu projeto para dar contexto ao assistente sobre a integração:

```markdown
# Regras do projeto — Integração Vendaze

## API

- Base URL: https://api.vendaze.com/v1
- Autenticação: Bearer token via OAuth 2.1
- Paginação: page + per_page, máximo 100 por página
- Valores monetários sempre em centavos

## MCP Server

- URL: https://api.vendaze.com/mcp
- Ferramentas disponíveis: list_people, get_person, create_person, list_deals, etc.
- Campos adicionais retornados apenas em get\_\* (não em listagens)

## Convenções de código

- Erros tratados pelo código: token_expired (renovar), rate_limit_exceeded (retry com backoff)
- Nunca expor client_secret no frontend
- workspace_id sempre do JWT, nunca do cliente
```

## Renovação de token

Para evitar que o token expire durante uma sessão de desenvolvimento longa, configure uma tarefa no `package.json` para renovar antes de iniciar:

```json
{
  "scripts": {
    "dev": "node scripts/renew-vendaze-token.js && next dev",
    "renew-token": "node scripts/renew-vendaze-token.js"
  }
}
```

O script de renovação segue o mesmo padrão descrito no [guia do Claude](/pt/ai/claude#renovação-de-token).
