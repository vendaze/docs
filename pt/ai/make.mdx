O Make (anteriormente Integromat) permite construir cenários de automação complexos com lógica condicional, transformação de dados e múltiplas integrações em série ou em paralelo. É uma opção poderosa para equipes que precisam de mais controle do que o Zapier oferece.

## Como conectar

### 1. Registrar o app no Vendaze

```bash
curl -X POST https://api.vendaze.com/v1/auth/register-app \
  -H "Content-Type: application/json" \
  -d '{
    "app_name": "Make Integration",
    "email": "dev@seuapp.com",
    "redirect_uris": ["https://www.make.com/oauth/cb/oauth2"],
    "scopes": ["people:read", "people:write", "companies:read", "companies:write", "deals:read", "deals:write", "tasks:read", "tasks:write"]
  }'
```

### 2. Criar a conexão no Make

1. Acesse [make.com](https://make.com) e faça login
2. Em um cenário, adicione um módulo **Vendaze**
3. Clique em **Add** para criar uma nova conexão
4. Insira o `client_id` e `client_secret` recebidos por email
5. Clique em **Save** — você será redirecionado para autorizar no Vendaze
6. Selecione o workspace e confirme os escopos

### 3. Módulos disponíveis

**Triggers (Watch):**

- Watch People — monitora novos contatos ou contatos atualizados
- Watch Companies — monitora novas empresas
- Watch Deals — monitora novos negócios ou negócios atualizados
- Watch Tasks — monitora novas tarefas

**Actions:**

- Create a Person
- Update a Person
- Get a Person
- Search People
- Create a Company
- Update a Company
- Search Companies
- Create a Deal
- Update a Deal
- Search Deals
- Create a Task
- Update a Task
- Log an Activity

**Iterators:**

- List All People — itera por todos os contatos
- List All Deals — itera por todos os negócios

## Exemplos de cenários

**Sincronização bidirecional:**
Quando um negócio é criado no Vendaze, cria um projeto no Asana. Quando o projeto no Asana é marcado como concluído, atualiza o negócio no Vendaze para o stage "Ganho".

**Enriquecimento de contatos:**
Quando um novo contato é criado no Vendaze, chama uma API de enriquecimento de dados, e usa o módulo **Update a Person** para preencher campos adicionais com as informações retornadas.

**Notificação de negócio parado:**
Verifica diariamente os negócios que não foram atualizados nos últimos 7 dias e envia uma mensagem no Slack para o responsável.

## Tratamento de erros no Make

O Make suporta tratamento de erros por módulo. Configure um handler de erro para lidar com respostas `429` (rate limit) usando a opção **Resume** com delay de 60 segundos.

Para erros `401`, configure um handler que renova o token de acesso via módulo HTTP antes de tentar novamente.

## Agendamento

Cenários com Watch modules verificam novos eventos no intervalo configurado (mínimo 1 minuto no plano Core). Para dados em tempo real, configure um webhook no painel do Vendaze apontando para o URL de entrada do Make.
